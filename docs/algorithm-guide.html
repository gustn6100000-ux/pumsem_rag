  "work_name": "콘크리트 타설",

3 검색 — searchPipeline → targetSearch

임베딩 생성 → 4단계 캐스케이드 검색 → 결과 분기

3-1. 임베딩 생성

질문 텍스트를 Gemini text-embedding-004 모델로 768차원 벡터로 변환합니다.

"콘크리트 타설 인력" → [0.023, -0.156, 0.089, ..., 0.042]  // 768차원

3-2. targetSearch — 4단계 캐스케이드 검색

정확도가 높은 순서대로 시도하며, 결과가 나오면 즉시 반환합니다:

1단계
ILIKE 정확 매칭
work_name + spec 조합으로
%콘크리트 타설%
2단계
키워드 ILIKE
키워드별 + 도메인 동의어
%콘크리트% OR %레미콘%
3단계
벡터 검색
pgvector 코사인 유사도
threshold ≥ 0.4
4단계
chunk 본문 검색
graph_chunks.text ILIKE
본문 텍스트 직접 검색
🆕 고아 섹션 감점 (신규 패치): 3단계 벡터 검색에서 chunk가 0건인 Section의 유사도를 -0.3 감산하여 후순위로 밀어냅니다. 이것이 "3-3 현장 타설 콘크리트 라이닝" 같은 빈 섹션 문제를 해결합니다.

3-3. 도메인 동의어 확장

4 명확화 — graphClarify / resolveSection

검색 결과가 여러 섹션에 걸쳐 있으면 사용자에게 선택 옵션을 제시합니다

resolveSection — 4가지 탐색 전략

전략방식설명
전략 1Section ILIKESection 이름으로 ILIKE 검색
전략 2WorkType ILIKEWorkType 이름으로 직접 검색
전략 3키워드 독립키워드별 독립 검색 후 병합
전략 4chunk 텍스트원문 본문에서 키워드 검색

계층 레벨 판정

레벨조건UI 동작
multi_section2개+ 섹션🏗️ 섹션 선택 칩 (분야 선택)
sub_section1섹션, 2개+ 분류📁 분류 선택 칩 (V형/H형 등)
worktype_many1섹션, 4개+ 작업📋 작업 선택 목록
worktype_few1~3개 작업💡 칩 + 자동 답변
empty작업 없음📄 "전체 내용 보기" 제시
🆕 고아 섹션 필터 (신규 패치): 계층 판정 시 chunkMeta에 없는 Section(= chunk 0건) 엔티티의 관련성 점수를 -100점 감점하고, clarify 옵션에서 빈 섹션을 제외합니다.

5-1. 그래프 확장 (expandGraph)

🌐 지식 그래프 탐색 예시
WorkType: "레디믹스트콘크리트 타설" (W-1282)
├── REQUIRES_LABOR → 콘크리트공 (인, 수량: 0.15~0.85)
├── REQUIRES_LABOR → 보통인부 (인, 수량: 0.46~1.36)
├── REQUIRES_EQUIPMENT → 굴착기 0.6~0.8m³ (대, 수량: 1)
├── USES_MATERIAL → 레디믹스트 콘크리트
└── HAS_NOTE → "소량 산재 시 50%까지 감"

5-2. 컨텍스트 조합 (buildContext)

수집된 데이터를 LLM이 이해할 수 있는 마크다운 형식으로 변환합니다.

교차표 형식으로 동일 직종의 조건별 수치를 비교 가능하게 렌더링합니다.

5-3. LLM 답변 생성 (generateAnswer)

컨텍스트를 DeepSeek v3.2에 전달하여 자연어 답변을 생성합니다.

비용 산정 시에는 labor_costs 테이블에서 2026년 노임단가를 주입합니다.

6 전체 내용 보기 — fullViewPipeline

사용자가 "전체 내용 보기"를 클릭하면 해당 섹션의 원문 + 그래프 데이터를 결합합니다

flowchart TD S["section_id 수신"] --> C["graph_chunks 로딩"] C --> SUB{"sub_section?"} SUB -->|"Yes"| F["해당 분류만 필터"] SUB -->|"No"| M["전체 chunk 병합"] F --> WT["WorkType 4단계 폴백 탐색"] M --> WT WT --> EX["expandGraph"] EX --> CTX["원문 + 그래프 = 컨텍스트"] CTX --> LLM["DeepSeek 답변"]

WorkType 4단계 폴백 탐색

단계방식설명
3-1직접 매칭동일 section_id의 WorkType 검색
3-2cross-reference동일 title의 다른 section에서 검색
3-3하위 절하위 section (prefix 매칭)에서 검색
3-4Section 자체Section 엔티티 expandGraph (최후 수단)
🆕 고아 섹션 폴백 (신규 패치): chunk가 없는 섹션에 진입하면, Section 이름의 키워드로 WorkType을 직접 ILIKE 재검색하여 answerPipeline으로 전환합니다.
